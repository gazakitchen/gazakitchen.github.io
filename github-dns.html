دليل ربط الدومين بمنصة غزة كيتشن

شرح سجلات A Records وسجلات CNAME وكيفية إضافتها

ما هي سجلات الـ DNS؟

هي بمثابة "دفتر عناوين" يخبر المتصفح أين يجد موقعك. لربط موقعك بـ GitHub، نحتاج لتوجيه الدومين إلى خوادم GitHub باستخدام طريقتين:

الخيار الأول: الأكثر شيوعاً

ربط دومين فرعي (Subdomain)

إذا كنت تريد رابطاً مثل check.yourdomain.com. هذه الطريقة هي **الأسهل والأكثر استقراراً**.

ماذا نكتب؟ نستخدم سجل CNAME فقط.

Type Name / Host Value / Target
CNAME check (أو أي اسم تختاره) username.github.io
الخيار الثاني

ربط الدومين الأساسي (Root Domain)

إذا كنت تريد الرابط أن يكون الدومين نفسه مباشرة مثل yourdomain.com.

ما هي سجلات A Records؟ هي سجلات تربط اسم الدومين مباشرة بعناوين IP الخاصة بخوادم GitHub. يجب إضافة 4 سجلات لضمان التوافر الدائم للموقع.

ماذا نكتب؟ نستخدم سجلات A.

Type Name / Host Value (IP Address)
A@185.199.108.153
A@185.199.109.153
A@185.199.110.153
A@185.199.111.153

* ملاحظة: الرمز @ في خانة الاسم يعني الدومين الأساسي نفسه.

⚠️ تنبيه هام: بعد إضافة السجلات في لوحة تحكم الدومين، يجب عليك كتابة الدومين (أو الدومين الفرعي) في حقل "الدومين الخاص" داخل لوحة التحكم عند الرفع ليتم إنشاء ملف CNAME تلقائياً.
